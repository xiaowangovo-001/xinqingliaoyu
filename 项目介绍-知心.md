# 知心 (ZhiXin) — 长时段心理健康的 AI 陪伴工具

> 一个记得住你每一次倾诉、能请来苏轼/王阳明/苏格拉底陪你聊天的 AI 情绪陪伴工具。

---

## 一、产品简介

**为什么做这个**

小红书月活用户中95后与00后合计占比达85%，一线及新一线城市用户占66%以上。然而这个群体正面临严峻的心理健康挑战：德勤2025年调查显示，40%的Z世代处于压力或焦虑状态，其中约三分之一认为工作为压力主要来源；《中国青年报》调查表明，80.22%的受访大学生存在不同程度的社交焦虑；《柳叶刀-区域健康》2025年发布的数据显示，中国儿童与青少年精神疾病整体患病率已达19.3%，其中焦虑症8.2%、抑郁症6.1%。

在此背景下，越来越多年轻人转向AI寻求情绪出口。复旦发展研究院《中国青年网民社会心态调查报告（2024）》显示，13.5%的年轻人选择向AI虚拟人倾诉心事，该比例已超过向父母亲人倾诉。《美国医学会杂志》（JAMA）研究亦指出，约每8名美国青少年与年轻成人中就有1人曾使用AI进行情绪咨询。AI陪伴因其24小时在线、无评判、高私密性等特征，正在成为传统社交支持体系之外的重要补充。

**当前产品的核心缺陷**

然而，市面上的AI陪伴产品普遍局限于"单次对话"——用户每次开启会话均需重新建立语境，系统缺乏对历史互动信息的调用与整合能力。相关研究指出，大语言模型在心理支持场景中的核心局限即在于其"无状态"（stateless）特性，导致连续对话中容易出现"过早推进、阶段错位和边界越界"等问题。此外，产品风格同质化严重，缺少文化厚度。

**知心的解法：两项核心创新**

一是**长时段情绪追踪**——通过记录与分析用户情绪的时间序列数据，生成可视化的成长轨迹（Dashboard），使AI从"即时回应工具"转化为"持续陪伴的见证者"，让用户"看见"自己的情绪变化与成长。

二是**智者对话模式**——当用户面临具体困惑时，AI可切换至苏轼（豁达）、王阳明（内省）、苏格拉底（诘问）三种历史智者的人格进行回应。这不是简单的"皮肤切换"，而是从底层System Prompt开始完全不同的对话哲学——苏轼用自嘲和自然比喻化解焦虑的重量感，王阳明以"知行合一"引导向内看，苏格拉底永远只问不答——以千年人文智慧拓展情绪陪伴的深度，同时回应市场上AI陪伴产品风格同质化的局限。

## 二、技术架构与使用工具

| 层级 | 技术选型 | 说明 |
|---|---|---|
| 前端框架 | Next.js 16 + TypeScript | 静态导出（SSG），零服务器成本 |
| AI 模型 | DeepSeek V4 Pro | 流式对话，中文理解与生成能力强 |
| 状态管理 | Zustand | 轻量无模板代码，AI 协作友好 |
| 样式方案 | Tailwind CSS 4 | 原子化CSS，迭代速度快 |
| 数据持久化 | localStorage | 纯前端方案，用户数据留在本地 |
| 数据可视化 | Recharts | 情绪趋势 Dashboard |
| 部署方案 | GitHub Pages + Actions | Push 自动构建发布，免费托管 |
| 数据库（预留）| Supabase | Schema 已设计，可随时接入云端同步 |

核心功能链路：落地页 → 简易登录 → 对话系统（4 种智者人格 × 流式 AI 响应 × 情绪标签）→ 聊天记录自动持久化与历史对话恢复 → 成长轨迹 Dashboard（情绪趋势 + 连续天数 + 里程碑 + 月度回顾）。目前已完成全部核心功能的 Demo 搭建。

## 三、与 AI 的协作方式

本项目 100% 通过 **Claude Code** 以自然语言对话完成开发。过程中无需编写任何代码，仅通过描述性指令驱动 AI 逐步搭建整个产品。总对话约 460 轮（用户消息 460+，AI 响应 880+），4 天从零到可部署产品。

**第一阶段：初步构思与定位**

使用 DeepSeek 等国产大模型进行开放式头脑风暴，围绕"心理疗愈 + 长时段思考"的核心方向，与模型进行多轮对话，逐步明确产品的功能定位、目标用户与核心交互逻辑。这一阶段重点借助 DeepSeek 的中文理解能力，梳理了"情绪记录→趋势分析→温暖回应"的产品逻辑链路，并初步设计了 AI 的人设方向与对话风格框架。

**第二阶段：功能生成与代码搭建**

将结构化的功能需求以自然语言形式输入 Claude Code，由其生成可运行的完整代码。过程中仅通过"这个按钮应该放在哪里""情绪图表用什么形式展示""侧边栏的历史对话要怎么恢复"等描述性指令，推动 Claude Code 逐步搭建出包括路由结构、组件树、状态管理、API 对接在内的交互原型主体架构。

**第三阶段：打磨、迭代与"AI训练"**

原型搭建完成后，进行多轮沉浸式自测，模拟真实用户使用场景，记录每一处交互体验上的不顺畅、AI 回应中的人设偏离、以及界面审美上的不足。之后将这些问题逐条反馈给 AI 工具——例如"这段回复的语气过于机械，能否加入更多共情的表达""情绪趋势图的视觉呈现不够直观"——通过反复对话与迭代优化，逐步将产品调校至可演示状态。这一过程本身即构成了对"AI训练师"岗位能力的直接演练：训练 AI 输出更贴合预期的人性化内容，并持续调校其表现风格与响应质量。

**关键协作案例**

- **System Prompt 设计**：将心理咨询中的"倾听-验证-重构-行动"框架与苏轼豁达哲学、王阳明心学（知行合一、致良知）、苏格拉底诘问法（产婆术）深度融合，为 4 种 AI 人格分别定义了对话哲学、语气风格、三段式回应结构和安全底线，总计约 8000 字——这正是"用心理学/哲学/历史视角定义 AI 输出标准"的直接实践。
- **API 故障诊断**：部署后出现 401 认证错误，Claude Code 自动添加 API 诊断端点定位到 GitHub Actions 构建时密钥文件被 .gitignore 排除的问题，3 轮对话完成修复。
- **聊天记录持久化**：从"刷新后消息消失"的用户体验问题出发，经 3 轮对话完成 5 个文件的修改（Store 持久化层 + Hook 元数据保存 + Sidebar 加载逻辑 + ChatInterface 自动恢复 + URL 参数支持），构建部署一步到位。
